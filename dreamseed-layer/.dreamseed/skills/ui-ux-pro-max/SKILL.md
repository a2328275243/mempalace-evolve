# UI/UX Pro Max

Use this skill when the task changes how an interface looks, feels, moves, or is interacted with.

## Fast Operating Rules

- Start with the existing product context, design system, framework, and audience.
- Prioritize accessibility, touch/keyboard interaction, responsive layout, typography, color contrast, performance, forms, navigation, animation, and chart clarity.
- Keep default reasoning compact. Do not paste the full database into the conversation unless the user explicitly asks for deep design guidance or the task needs it.
- For detailed rules, palettes, product-type guidance, stack-specific examples, or search commands, read `references/full.md` for only the relevant section.

## Must Use

- Creating or refactoring UI components, pages, dashboards, landing pages, SaaS/admin surfaces, e-commerce, portfolios, blogs, or mobile screens.
- Reviewing UI code for UX, accessibility, visual consistency, or professional polish.
- Choosing color, type, spacing, layout, motion, navigation, forms, tables, cards, modals, charts, or interaction states.

## Skip

- Pure backend/API/database work.
- Infrastructure, DevOps, non-visual scripts, or performance work unrelated to UI.
